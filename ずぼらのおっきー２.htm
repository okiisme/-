どこのサイトにアップロードすればいいですか？

おっきーさんのウェブサイトを無料でホスティングするために、以下のサービスを検討してみてください。これらのプラットフォームは初心者でも使いやすく、簡単にサイトを公開することができます。

1. GitHub Pages

GitHub Pagesは、GitHubリポジトリから直接ウェブサイトをホスティングする無料サービスです。静的サイト（HTML、CSS、JavaScript）に最適です。

手順：

  1. GitHubアカウントを作成します。
  2. 新しいリポジトリを作成し、index.htmlとstyle.cssをアップロードします。
  3. リポジトリ設定で「GitHub Pages」を有効にし、公開します。

参考リンク：

  • GitHub Pagesの公式ドキュメント

2. Netlify

Netlifyは、静的サイトのデプロイとホスティングを簡単に行えるプラットフォームです。カスタムドメインや自動SSL証明書も提供しています。

手順：

  1. Netlifyアカウントを作成します。
  2. 新しいサイトを作成し、GitHubリポジトリからデプロイするか、直接ファイルをアップロードします。

参考リンク：

  • Netlifyの公式サイト

3. Vercel

Vercelは、静的サイトおよびサーバーレス機能を提供するホスティングサービスです。GitHub、GitLab、Bitbucketと統合しており、簡単にデプロイできます。

手順：

  1. Vercelアカウントを作成します。
  2. 新しいプロジェクトを作成し、GitHubリポジトリからインポートします。

参考リンク：

  • Vercelの公式サイト

4. GitLab Pages

GitLab Pagesは、GitLabリポジトリから静的サイトをホスティングできるサービスです。GitLab CI/CDを使って自動デプロイが可能です。

手順：

  1. GitLabアカウントを作成します。
  2. 新しいリポジトリを作成し、.gitlab-ci.ymlを設定してサイトをデプロイします。

参考リンク：

  • GitLab Pagesの公式ドキュメント

具体的なアップロード例（GitHub Pagesの場合）

1. GitHubアカウント作成

  • GitHubにサインアップ

2. リポジトリの作成

  1. GitHubでログインし、右上の「+」アイコンから「New repository」を選択。
  2. リポジトリ名を入力し、「Create repository」をクリック。

3. ファイルのアップロード

  1. 作成したリポジトリに移動し、「Add file」→「Upload files」を選択。
  2. index.htmlとstyle.cssをドラッグ&ドロップでアップロードし、「Commit changes」をクリック。

4. GitHub Pagesの有効化

  1. リポジトリの「Settings」に移動。
  2. 左側のメニューから「Pages」を選択し、「Source」セクションで「main branch」を選択して保存。
  3. しばらく待つと、指定されたURLでサイトが公開されます。
